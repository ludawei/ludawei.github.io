MKMapView经纬度引起的一个bug

2016年06月02

解决方法：

if (anno.coordinate.longitude == 180)
{
    anno.coordinate = CLLocationCoordinate2DMake(anno.coordinate.latitude, -180);
}

原因应该是MKMapView默认的 纬度 范围是（-180 ~ 180），包含-180，但不包含180，故认为180的纬度是无效的。 解决方法就是将180变为-180（因为两者是相等的）
